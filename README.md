# Mocha Tracking → Excel Converter

Automates the extraction and processing of motion-tracking keyframe data from `.mocha` export files generated by Mocha Pro and converts them into structured coordinate tables using an Excel template.

The script reads Mocha tracking exports, parses translation keyframes for multiple layers, feeds the data into a predefined Excel template, triggers spreadsheet formulas, and produces processed `.xlsx` files containing X/Y coordinate data.

This replaces a manual workflow where tracking data must be copied into Excel, split into columns, processed with formulas, and organized into usable coordinate sets.

---

# Overview

Motion tracking software exports tracking data as text containing keyframe definitions. These lines typically look like:


Layer_1.Track.Translation_X.keyframes.append( Key(...)
Layer_1.Track.Translation_Y.keyframes.append( Key(...)


This script:

1. Finds `.mocha` files in the script directory
2. Copies an Excel template
3. Extracts relevant keyframe lines from the Mocha export
4. Inserts the data into the template
5. Uses Excel formulas to parse and calculate coordinate values
6. Writes the final coordinate results into dedicated columns
7. Saves the processed workbook as a new `.xlsx` file

Each `.mocha` file becomes a corresponding Excel file with structured tracking data.

---

# Features

- Automated parsing of `.mocha` tracking exports
- Processes multiple files in batch
- Supports up to **4 tracked layers**
- Uses an Excel template to perform coordinate calculations
- Automatically splits and processes keyframe strings
- Generates clean `.xlsx` files with extracted coordinates
- Eliminates manual copy/paste workflow

# How It Works

## 1. Input Discovery

The script scans its working directory for:

- `.mocha` tracking files
- `template.xlsx`


project/
├── analysis.py
├── template.xlsx
├── shot01.mocha
├── shot02.mocha


---

## 2. Template Duplication

For each `.mocha` file:


shot01.mocha → shot01.xlsx


A copy of the Excel template is created to store the processed results.

---

## 3. Tracking Data Extraction

The script searches the `.mocha` file for lines containing:

- control point identifiers
- translation X keyframes
- translation Y keyframes

Example markers:


Layer_1.Track.Translation_X.keyframes.append(
Layer_1.Track.Translation_Y.keyframes.append(


These lines contain frame-by-frame tracking coordinates.

---

## 4. Excel Data Processing

Using the Windows COM interface (`win32com`), the script controls Excel to:

- insert extracted lines into specific columns
- run **Text-to-Columns** parsing
- trigger spreadsheet formulas
- calculate coordinate values

The Excel template performs operations such as:

- parsing frame numbers
- extracting translation values
- scaling and offsets
- coordinate cleanup

---

## 5. Output Generation

After Excel calculations complete, the script copies the final coordinate columns to designated output fields.

Each processed `.mocha` file produces a finished Excel workbook containing structured tracking coordinates.

---

# Supported Tracking Layers

The script is configured to process **four layers**:

| Layer | Control Point |
|-----|-----|
| Layer 1 | Spline 1 Control Point |
| Layer 2 | Spline 2 Control Point |
| Layer 3 | Spline 3 Control Point |
| Layer 4 | Spline 4 Control Point |

Layer processing is defined in the `LAYER_CONFIG` configuration block.

# Configuration

Tracking layer behavior is defined in:


LAYER_CONFIG


Each layer entry specifies:

- control point identifier
- X translation marker
- Y translation marker
- source formula columns
- destination output columns

This allows the script to adapt to different Mocha export formats.

# Limitations

- Requires **Microsoft Excel installed locally**
- Runs only on **Windows** due to COM automation
- Depends on the structure of the provided Excel template
- Assumes Mocha export syntax remains consistent
